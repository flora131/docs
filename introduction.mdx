# Atomic documentation

Atomic helps software teams turn one-off coding-agent sessions into repeatable engineering workflows. It combines a full terminal coding agent with deterministic orchestration, reusable skills, specialized subagents, MCP tools, web research, session management, and a TypeScript workflow SDK.

## When to use Atomic

Use Atomic when a task needs more structure than a single chat turn:

- large codebase research before implementation
- repeatable debug → fix → review loops
- migrations that need staged validation
- autonomous work that must pause for human approval
- coding-agent workflows your team wants to version with the repo
- custom internal tools that should be callable from the agent

## Mental model

Atomic has three layers:

1. **The agent session** — the interactive or print-mode coding agent that reads files, edits code, runs commands, and uses tools.
2. **Resources** — settings, skills, extensions, prompt templates, themes, MCP servers, custom tools, and packages discovered from user and project directories.
3. **Workflows and subagents** — orchestration primitives that launch one or more agent sessions with explicit inputs, context handoffs, gates, and validation.
