# Quickstart

## 1. Install Atomic

Install the CLI globally with your package manager of choice:

<Tabs>
  <Tab title="npm">
    ```bash
    npm install -g @bastani/atomic
    ```
  </Tab>
  <Tab title="pnpm">
    ```bash
    pnpm add -g @bastani/atomic
    ```
  </Tab>
  <Tab title="bun">
    ```bash
    bun add -g @bastani/atomic
    ```
  </Tab>
</Tabs>

Then verify the command is available:

```bash
atomic --help
```

## 2. Authenticate a provider

Atomic can use API-key providers and subscription providers.

### Option A: use an API key

```bash
export ANTHROPIC_API_KEY=sk-ant-...
atomic
```

Common environment variables include `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `GOOGLE_GENERATIVE_AI_API_KEY`, `OPENROUTER_API_KEY`, and cloud-provider-specific variables for Azure OpenAI, Bedrock, Vertex AI, and Cloudflare.

### Option B: sign in from the TUI

Start Atomic and run `/login`:

```bash
atomic
```

```text
/login
```

Then choose a provider such as Claude Pro/Max, ChatGPT Plus/Pro, or GitHub Copilot.

<Info>
  See [Providers](/reference/providers) for the full provider list, auth-file behavior, and cloud-provider setup.
</Info>

## 3. Start your first session

Run Atomic inside a project directory:

```bash
cd /path/to/your/repo
atomic
```

Try a prompt that gives the agent a concrete goal:

```text
Read the README and package files, then summarize how this project is organized. Do not edit files yet.
```

Atomic will inspect the repo, use tools, and stream its reasoning and results in the terminal UI.

## 4. Give the project instructions

Add an `AGENTS.md` or project settings file so every session starts with the right conventions:

```md AGENTS.md
# Project instructions

- Use bun for package management.
- Run `bun test` before claiming code changes are complete.
- Prefer small, focused edits.
- Do not modify generated files under `dist/`.
```

Atomic discovers context files and project settings automatically; see [Configuration](/getting-started/configuration) and [Settings](/reference/settings).

## 5. Try non-interactive mode

For scripts and CI-like checks, use print mode:

```bash
atomic -p "Explain the public API of this package"
```

You can also pipe stdin:

```bash
git diff --staged | atomic -p "Review this diff for correctness issues"
```

## 6. Run a workflow

List installed workflows:

```text
/workflow list
```

Run a workflow with named inputs:

```text
/workflow deep-research-codebase prompt="Map how authentication works in this repo"
```

For long-running autonomous workflows, use a devcontainer, VM, or remote development machine.

<Warning>
  Workflows are designed to keep multi-step pipelines moving. Treat autonomous workflow execution as powerful automation: run it in an isolated development environment when it can edit files or run commands unattended.
</Warning>
