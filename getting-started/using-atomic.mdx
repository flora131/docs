# Using Atomic

Atomic can be used as an interactive terminal application, a one-shot CLI, an SDK, or the execution engine underneath workflows.

## Interactive mode

Start the terminal UI with:

```bash
atomic
```

In interactive mode you can:

- ask the agent to inspect, edit, and test your code
- paste images or reference files where supported by your terminal
- run slash commands such as `/login`, `/model`, `/tree`, `/workflow`, and `/help`
- switch models and thinking levels without restarting
- continue, fork, clone, name, export, and share sessions
- use project resources such as skills, prompt templates, extensions, and MCP tools

## Good prompts

Atomic works best when you give it a target, constraints, and a validation expectation:

```text
Find where request retry logic is implemented. Explain the flow with file paths and line numbers. Do not edit files.
```

```text
Fix the failing auth test. Run the smallest relevant test first, then the full package test if the fix is isolated. Summarize changed files and commands run.
```

```text
Use the research-codebase skill to map the billing webhook flow and write a research doc. Do not implement anything yet.
```

## Slash commands

Type `/` in the TUI to discover available commands. Important commands include:

| Command | Purpose |
| --- | --- |
| `/help` | Show command help. |
| `/login` | Authenticate subscription-based providers. |
| `/model` | Select a model. |
| `/thinking` | Change thinking level when the model supports it. |
| `/sessions` | Resume or manage prior sessions. |
| `/tree` | Navigate a session's branch tree. |
| `/fork` | Create a branch from a prior point. |
| `/clone` | Copy a conversation branch. |
| `/compact` | Manually compact context. |
| `/skill:<name>` | Explicitly invoke a skill. |
| `/workflow ...` | List, run, pause, resume, and inspect workflows. |
| `/run`, `/parallel`, `/chain` | Launch subagents and subagent chains when the subagents package is available. |

Extensions and packages can register additional commands.

## Context files

Atomic loads instructions and resources from user and project locations. Common project-level inputs include:

- `AGENTS.md` or equivalent project instruction files
- `.atomic/settings.json` for project settings
- `.atomic/skills/**/SKILL.md` for project skills
- `.atomic/extensions/**/*.ts` for project extensions
- `.atomic/prompts/**/*.md` for reusable prompt templates
- `.atomic/themes/*.json` for themes

Use explicit file references in prompts when precision matters:

```text
Read src/server/auth.ts and test/auth.test.ts. Then explain the failing path before editing.
```

## Sessions

Atomic stores sessions so you can resume work later. A session is a tree, not only a flat transcript: you can fork from earlier points, label entries, and export conversations.

Common workflows:

- name important sessions with `/name`
- use `/tree` to branch before risky changes
- use `/fork` when you want an alternate approach
- use `/compact` when a long session needs a summary handoff
- export HTML for handoffs or review

See [Sessions](/reference/sessions) and [Session format](/reference/session-format) for details.

## Print mode

Use `-p` for non-interactive output:

```bash
atomic -p "Summarize this repository"
```

Print mode is useful for scripts, review pipelines, and quick questions. Combine it with shell input:

```bash
git diff --staged | atomic -p "Review this staged diff. Return blockers first."
```

## Model selection

Models are resolved from built-in providers, configured custom providers, auth state, and CLI overrides. Examples:

```bash
atomic --model anthropic/claude-sonnet-4
atomic --model openai/gpt-5.1 -p "Review this API design"
```

Inside the TUI, use `/model` and `/thinking`.

## Safety habits

- Ask for research first when the codebase is unfamiliar.
- Tell the agent when not to edit.
- Keep autonomous workflows inside an isolated environment.
- Require validation evidence for code changes.
- Use subagents for independent research/review instead of stuffing all context into one session.
